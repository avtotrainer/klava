# 01 — main.py

## როლი

`main.py` არის **ერთადერთი შესვლის წერტილი** (entry point) KLAVA პროექტში.

მისი ამოცანა არ არის:
- ლოგიკის გამოთვლა
- UI-ის მართვა
- სავარჯიშოების კონტროლი

მისი ამოცანაა მხოლოდ:
- გარემოს ინიციალიზაცია
- ფანჯრის შექმნა
- Trainer-ის გაშვება

---

## რატომ ცალკე main.py

`main.py` არსებობს იმისთვის, რომ:

- პროექტს ჰქონდეს **ერთი სტაბილური გაშვების წერტილი**
- არქიტექტურა იყოს მკაფიო და გასაგები
- შემდგომში შესაძლებელი იყოს:
  - ტესტირება
  - პაკეტიზაცია
  - გაშვება სხვადასხვა გარემოში

---

## პასუხისმგებლობები (Does)

`main.py`:

- ქმნის `tk.Tk()` ობიექტს
- აყენებს აპლიკაციის სათაურს
- გადასცემს root-ს Trainer-ს
- იძახებს `mainloop()`-ს

---

## რაც არ უნდა გააკეთოს (Does not)

`main.py` არ უნდა:

- შეიცავდეს ბიზნეს ლოგიკას
- იმპორტავდეს `logic/*` მოდულებს
- მართავდეს კლავიატურის მოვლენებს
- იცოდეს სავარჯიშოების შინაარსი
- შეიცავდეს UI ელემენტებს

თუ აქ ჩნდება ლოგიკა — არქიტექტურა დარღვეულია.

---

## დამოკიდებულებები (Imports)

`main.py` დამოკიდებულია მხოლოდ:

- `tkinter`
- `Trainer` კლასზე (`trainer.py`)

ყველაფერი სხვა არის Trainer-ის პასუხისმგებლობა.

---

## მინიმალური სტრუქტურა

სწორი `main.py` არის **მინიმალისტური**:

- მაქსიმალურად მოკლე
- მარტივად წასაკითხი
- პროგნოზირებადი

---

## მაგალითი (საბაზო ეტალონი)

```python
# main.py
# KLAVA — Entry Point

import tkinter as tk
from trainer import Trainer


def main():
    root = tk.Tk()
    root.title("KLAVA")

    Trainer(root)

    root.mainloop()


if __name__ == "__main__":
    main()
